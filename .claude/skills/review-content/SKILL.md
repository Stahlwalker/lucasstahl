When reviewing content, always:

1. **Read the target content** - Get the file path from the user or prompt for it
2. **Load the style guide** - Read `docs/Lukes-Developer-Writing-Style.md`
3. **Analyze thoroughly** for:
   - Tone and voice consistency
   - Formatting adherence
   - Structure and flow
   - Grammar and clarity
   - Style guide compliance
4. **Provide specific feedback** with line numbers and examples
5. **Give actionable suggestions** showing before/after examples

## Output Format

- **Overall Assessment**: Brief statement on style guide alignment
- **Strengths**: What's working well (be specific)
- **Issues Found**: Problems with line/section references
- **Suggestions**: Actionable improvements with examples
- **Style Guide References**: Which specific guidelines apply

Keep feedback constructive and specific. Always cite the style guide when making recommendations.
